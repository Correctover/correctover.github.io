Introduction

Correctover is a MCP Reliability Layer that validates LLM outputs in real-time before execution. It ensures that agent actions satisfy formal constraints, preventing failures before they happen.

Core Invariant

Required(τ) ⊆ Supported(τ)

For any task τ, all outputs the agent needs must be within what the model can produce.

Key Features

Feature Description
14.5μs latency Validation completes before tool execution begins
93 fault patterns Derived from 80K real API traces
Zero-config Drop-in replacement for OpenAI/Anthropic SDKs
Framework adapters CrewAI, AutoGen, LangGraph, Semantic Kernel

Quickstart

Install via pip:

pip install correctover

Use as a drop-in replacement:

from correctover import Correctover

client = Correctover()

# Same API as OpenAI, but with validation
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Hello"}],
    constraints={"max_tokens": 100, "temperature": 0.7}
)

# Output is validated before returning
print(response.choices[0].message.content)

Architecture

Correctover consists of five components:

1. Core Engine

Runtime validation engine. Handles constraint checking, recovery decisions, and state management.

2. Rules Library

93 fault patterns formalized from 80K real API traces. Covers rate limits, overloads, malformed outputs.

3. LocalGateway

Drop-in replacement for OpenAI/Anthropic SDKs. Runs locally with zero configuration.

4. Validator Plugin System

Extensible validation plugins. Write custom validators in Python/TypeScript.

5. Adapter Ecosystem

Pre-built adapters for popular agent frameworks. One-line integration.

Required(τ) ⊆ Supported(τ)

The core invariant of Correctover. For any task τ:

  • Required(τ): outputs the agent needs to proceed
  • Supported(τ): outputs the model can produce

If Required(τ) is not a subset of Supported(τ), the validation fails and recovery is triggered.
